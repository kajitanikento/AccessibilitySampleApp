# WCAG 2.2 準拠ルール

参照: https://waic.jp/translations/WCAG22/
対象レベル: **A / AA**

---

## 対象基準一覧（A / AA）

### 1. 知覚可能（Perceivable）

| 基準 | レベル | 要件 |
|---|---|---|
| 1.1.1 非テキストコンテンツ | A | 意味のある非テキスト要素に代替テキストを提供する。装飾要素は読み上げから除外する |
| 1.2.1 音声のみ・映像のみ | A | 音声のみ / 映像のみのコンテンツに代替手段を提供する |
| 1.2.2 キャプション（収録済み） | A | 動画にキャプションを提供する |
| 1.2.3 音声解説または代替メディア | A | 動画の視覚情報を説明する代替手段を提供する |
| 1.3.1 情報及び関係性 | A | 見出し・ラベル・グループの構造をプログラム的に識別できるようにする |
| 1.3.2 意味のある順序 | A | コンテンツの読み上げ順序が視覚的な意味と一致する |
| 1.3.3 感覚的な特徴 | A | 形・色・位置などの感覚情報だけで操作を説明しない |
| 1.3.4 画面の向き | AA | 縦横どちらの向きでも機能する |
| 1.3.5 入力目的の特定 | AA | 入力フィールドの目的（氏名・メール等）をプログラム的に示す |
| 1.4.1 色の使用 | A | 色だけに依存して情報を伝えない |
| 1.4.3 コントラスト（最低限） | AA | 通常テキスト 4.5:1 以上、大きいテキスト 3:1 以上 |
| 1.4.4 テキストのサイズ変更 | AA | テキストを 200% に拡大しても情報・機能が損なわれない |
| 1.4.10 リフロー | AA | 320px 幅相当でも横スクロールなしにコンテンツを表示できる |
| 1.4.11 非テキストコントラスト | AA | UIコンポーネント・アイコンが背景に対して 3:1 以上のコントラストを持つ |

### 2. 操作可能（Operable）

| 基準 | レベル | 要件 |
|---|---|---|
| 2.1.1 キーボード | A | 全機能をキーボード（Switch Control 含む）で操作できる |
| 2.1.2 キーボードトラップなし | A | キーボードフォーカスが特定の要素に閉じ込められない |
| 2.2.1 タイミング調整可能 | A | 時間制限がある場合は延長・停止・無効化できる |
| 2.4.1 ブロックのスキップ | A | 繰り返しコンテンツをスキップする手段を提供する |
| 2.4.2 ページタイトル | A | 画面に適切なタイトルを付ける |
| 2.4.3 フォーカス順序 | A | フォーカス移動が意味のある順序である |
| 2.4.4 リンクの目的 | A | リンク・ボタンの目的が名前から判断できる |
| 2.4.6 見出しとラベル | AA | 見出しとラベルがコンテンツを説明している |
| 2.4.11 フォーカスの非遮蔽（最低限） | AA | フォーカスされた要素がスティッキー UI に完全に隠れない ★2.2新規 |
| 2.5.1 ポインタジェスチャー | A | 複数指・軌跡ジェスチャーに単純な代替操作を提供する |
| 2.5.2 ポインタキャンセル | A | 操作を Up イベントで完了させ、誤操作をキャンセルできるようにする |
| 2.5.3 ラベルを名前に含める | A | 視覚ラベルのテキストをアクセシビリティ名に含める |
| 2.5.7 ドラッグ操作 | AA | ドラッグ操作に単一ポインタの代替手段を提供する ★2.2新規 |
| 2.5.8 ターゲットサイズ（最低限） | AA | タップターゲットを最低 24×24pt 確保する ★2.2新規 |

### 3. 理解可能（Understandable）

| 基準 | レベル | 要件 |
|---|---|---|
| 3.1.1 ページの言語 | A | コンテンツの言語をプログラム的に識別できるようにする |
| 3.2.1 フォーカス時 | A | フォーカスだけで予期しない状態変化を起こさない |
| 3.2.2 入力時 | A | 入力・選択だけで自動的に画面遷移しない |
| 3.2.3 一貫したナビゲーション | AA | ナビゲーションの位置・順序を全画面で統一する |
| 3.2.4 一貫した識別 | AA | 同じ機能を持つ要素を全画面で同じ方法で識別する |
| 3.3.1 エラーの特定 | A | 入力エラーをテキストで具体的に通知する |
| 3.3.2 ラベルや説明 | A | 入力フィールドにラベルまたは説明を提供する |
| 3.3.3 エラーの修正提案 | AA | エラー時に修正方法を提示する |
| 3.3.7 冗長な入力 | AA | 同一セッションで入力済みの情報を再入力させない ★2.2新規 |
| 3.3.8 アクセシブルな認証（最低限） | AA | 認知機能テストなしの認証手段を提供する ★2.2新規 |

### 4. 堅牢（Robust）

| 基準 | レベル | 要件 |
|---|---|---|
| 4.1.2 名前・役割・値 | A | 全インタラクティブ要素に名前・役割・状態を提供する |

> **注:** 4.1.1 構文解析は WCAG 2.2 で削除。ネイティブアプリは元々対象外のため影響なし。

## 優先度の考え方

仕様・実装を検討するときは以下の順で対応を判断する:

1. **レベル A** — 必ず満たす。未対応はアクセシビリティの根本的な欠如
2. **レベル AA** — 必ず満たす。日本のアクセシビリティ法令・調達基準の対象
3. **WCAG 2.2 新規基準**（★マーク）— 2.1 からの変更点として特に意識する
4. **レベル AAA** — 対応できれば対応する（このアプリでは任意）
