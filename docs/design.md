# SwiftUI Accessibility Sample App 設計

## 目的

SwiftUI アプリにおけるアクセシビリティ実装を、  
**WCAG 2.2 の A / AA レベルを基準として理解・共有するためのサンプルアプリ**を作成する。

対象:

- iOSアプリのデザイナー
- iOSエンジニア
- アクセシビリティレビュー担当

このアプリは次を目的とする。

- WCAG成功基準とUI実装の対応を理解する
- VoiceOver・音声コントロールなどの支援技術の挙動を体験する
- アクセシビリティ設計の共通理解を作る

UIおよび読み上げは **日本語を前提**とする。

## 画面構成

### WCAG項目一覧

各 WCAG 成功基準を一覧表示する。
各行に **項目名・レベル・概要説明** を表示し、詳細への入口とする。
一覧画面ではデモは表示しない。

### WCAG項目詳細

選択した成功基準の詳細を表示する。
**デモ**（インタラクティブな実装例）をこの画面に配置する。
デモがない項目は TODO として記載する。
ヘッダーには成功基準番号・適合レベル・概要説明を表示する。

# 4 堅牢

---

## 4.1.2 名前・役割・値 (A)

### 一覧（概要）

UI要素には適切な名前、役割、状態を提供する。

### 詳細（デモ）

- `accessibilityLabel` でラベルを付与して読み上げを確認する
- `accessibilityValue` で状態値を読み上げる
- `accessibilityHint` で操作方法を案内する
